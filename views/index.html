Our Process

We build our MVPs like this:

  1. Understand the problem

    We understand the problem that has been identified and confirm the intended users feel the pain and need for a solution.

  2. Build the solution

    We build the solution following the agile/lean software development best practices and get feedback from users as fast as possible.

  3. Review feedback

    We review client feedback and confirm we are making progress toward solving the problem.
